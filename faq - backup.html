FAQs

  • IT Service Management
    • Incident Management

      How will the Major Incident process be handled outside BMC?

      Major Incident Workbench are inherent in ServiceNow

      Will there be the option to ‘Convert INC to WO’?

      You cannot convert an INC to a WO in ServiceNow but, INC can be categorized as WO in the "Resolution Code" Field

      Currently all Linked Records are added in the same Relationships Tab. What will Related Records look like in the future?

      There are separate fields and tabs for Related Records (Parent/Child INCs, Problem, Caused by CRQ, Fix CRQ)

      The current Incident Priority is: Critical, High, Medium, Low. Will Incident Priority be the same in ServiceNow?

      In ServiceNow there will be a new ‘Planning’ Priority for Low Impact and Low Urgency Incidents

      Will ServiceNow be able to communicate with TFS? Incident, Work Orders and Notifications from other applications will be possible to redirect right to TFS?

      We are still exploring the possibility of creating/licensing a solution that can fill the current ADO/TFS integration need. We will have more information in the coming weeks on potential timing.

      Is there any common guidance on how SLA should be tagged with each form, because every EMIT group working with MSP are doing this differently today, so curious, if there is any common guidance which all SDO's can leverage?

      The SDO’s will update the SLA required to be setup in the SDO on boarding template. Only those SDO specific SLA’s will be configured in ServiceNow. If not updated in the SDO on boarding template, then they will follow their enterprise standard SLA’s.

      During the phased roll out, will it be ensured that one team using BMC can still transfer tickets to another using ServiceNow?

      The e-bonding solution is in place to help address teams that are in BMC interacting with teams that have migrated to ServiceNow. Tickets in BMC and ServiceNow will have some fields along with links to each other updated.

    • Problem Management

      Currently I am unable to flag Problems with inconclusive Root Cause Analysis, will this be the same ServiceNow?

      In ServiceNow there is an ‘Accept Risk’ functionality to identify Problems with no permanent resolution.

      At this time all investigation results are entered in the same Work Details Tab, how will investigation results be entered in ServiceNow?

      Each “Sequence of Events” is captured separately from “Analysis Info” (3 Steps: Workaround, Root Cause and Fix Notes).

      Configuration Item (CI) are not required will they be required in ServiceNow?

      Yes, Configuration Item (CI) are required in ServiceNow, the same as for Incident Management.

    • Change Management

      What will be the Risk Categories in ServiceNow?

      In ServiceNow there are 3 Risk Categories: Low, Moderate, High.

      For normal changes there is a minimum 7 days lead time, will that change?

      Yes, the lead time for normal changes will be flexible in ServiceNow.

      Currently changes are classified in 4 classes: Normal, Expedited, Standard, Emergency, what are the change classes in ServiceNow?

      In ServiceNow changes are classified in 3 classes: Normal, Standard, Emergency.

      Emergency changes can be approved after implementation, will that remain the same in ServiceNow?

      Emergency changes can only be used in very specific situations: Critical/high incidents, Implement ≤ 24 hours, and with approvals prior to implementation

    • Forms and User Portal

      Forms are currently created by the BMC centralized tools team; will they continue to create forms?

      No, catalog Items (forms) will be created and maintained by the process teams in ServiceNow

      Will changes to the IT Services Portal be segregated from changes to the services catalog?

      There will be an integrated portal & service offering in ServiceNow. Once a new catalog item is created it’s automatically available in the Self-Service portal

      Currently there is one single portal and service catalog for all users, will this change?

      Yes, there will be different Service Catalogs

      In BMC, Service Requests are available to create work orders and assign them to specific queues. How will that change in ServiceNow?

      Service Requests from BMC will be replicated in ServiceNow. Request tickets will replace work orders

  • IT Operations Management
    • Reporting and Analytics

      Currently all reports & dashboards are custom and built on ITDW, will this change?

      Yes, ServiceNow offers a wealth of Out of The Box reports, dashboards and KPIs. All reports and dashboards are created directly in ServiceNow (for data sourced in ServiceNow).

      Operational data from BMC is currently sent to ITDW for consumption, will this process change?

      No, operational data from ServiceNow will be sent to ITDW for consumption.

      How can we retrieve older ticket/request information when ITSM is gone?

      BMC Remedy will be set to Read-Only as teams are on-boarded to ServiceNow. Older information will be available in the ITDW and we will be retaining the data set in an offline mode for the three-year retention period.

    • Data including CMDB

      The CMDB interface is outdated, will the ServiceNow interface be better?

      The ServiceNow interface is intuitive and easy to use

      The current system uses centralized discovery and there is a lack of consistent data ownership; how will this be improved?

      We are moving to a federated discovery and there will be a comprehensive data governance with ServiceNow

      Will the current product catalog be loaded into Service Now as is?

      We extracted a copy of all BMC forms and are reviewing each one of them. To avoid a GIGO (garbage in garbage out) situation, we'll be applying a governance process to ensure clear ownership & active forms are migrated.

      As the forms/user portal are being changed to the ESM tool, is there a significant difference from a User Experience? Do we assume Business users can adopt the new User Portal easily?

      We anticipate some slight differences because the ServiceNow platform is different from the current CMS platform we are using. Our UI/UX team on the project is working to ensure the changes are minimal.

      Will users be able to update CMDB via Service Now?

      Like the BMC CMDB, some fields will be locked down because the data is discovered while other fields may be updated manually by authorized personnel. The security model for the ServiceNow CMDB is still under development.

  • ServiceNow Platform
    • Platform Architecture & Support

      BMC is an On-Premise, centralized support, highly customized system, how is ServiceNow different?

      ServiceNow is a Cloud SaaS solution with Self-Service & Support provided by MSP, utilizing Out of The Box functionality.

      Are we aware of any 'sensitive' data issues we need to address with moving to the cloud and support to an MSP?

      There will be no sensitive data in the ServiceNow platform. No proprietary or private classified data.

    • APIs

      What is the plan for the ITSM API transition?

      The transition will be worked as part of the on-boarding process, but the API for ServiceNow is available now. Please find additional information at this link.

      Can I continue to use the pre-existing Ticketing API on-premise?

      No. Will need to update to the ServiceNow API which requires a direct connection to ServiceNow (Cloud) and oAuth (basic authentication requires an exception).

      Where can I go to learn more about port openings (direct connection to ServiceNow is required)?

      Our internal ServiceNow Wiki has details on what information to include in the port opening request and how to submit the request.

      Where can I learn more about oAuth?

      The MAP team has published information regarding the oAuth authentication method on their wiki. In cases that you need more assistance, a request can be submitted at goto/appconsult.

      Are there reference materials for learning ServiceNow API?

      • REST API Explorer: You can browse available APIs, API versions, and methods for each API. Just Navigate to System Web Services > REST API Explorer.
      • How to create the encoded query: You can generate an encoded query string through a filter on any list and paste the string into a URL query or a reference qualifier.
      • Incident on Development environment: Table information from the instance is used to provide a list of endpoints, methods, and variables.
    • Notable System Integrations

      Will Service Now be able to communicate with TFS? Incident, Work Orders and Notifications from other applications will be possible to redirect right to TFS?

      We are still exploring the possibility of creating/licensing a solution that can fill the current ADO/TFS integration need. We will have more information in the coming weeks on potential timing.

      Why are we doing a waterfall roll-out approach?

      We chose to roll SDO’s out in batches to manage the impact on system and project resources as we go through the on boarding process. During this transition, we have put a 2-way integration in place (eBonding) between our legacy ITSM system and ServiceNow to ensure tickets are correctly routed.

      How will ServiceNow handle account credentials from a security perspective?

      User access to the system is controlled via the company mandated Single Sign On process. You will automatically have access once your roles are assigned. Integration credentials will be managed securely through XOM’s password vaulting system, CyberArk.

      Will each team have to build their own automation capabilities?

      There will be re-usable automation use cases built on the platform that can be leveraged by distributed teams.

      How will the teams be able to leverage centralized automation use cases offered by the platform?

      There will be a centralized intake process for submitting the demand or idea requests. The governance of these requests reviews the design templates and helps leverage re-usable components offered by the platform.
