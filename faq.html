  • Incident Management

    How is the contact type field used?

    Contact type is used to indicate "How" the incident was reported.

    How can you view tasks associated with an incident?

    You can scroll down on the incident page and find a tab “Incident Tasks” which list the tasks associated with the Incident ticket. You can also add or modify any tasks in the incident ticket.

    What is the best practice to handle cases where incidents were previously converted to work orders?

    Resolve the Incident and categorize it as "User Request/Information/How-To” in the ‘Resolution Code’ field located in the Resolution Information Tab

    If a High or Critical incident has breached the SLA, will the ticket escalate automatically?

    There is some capability to trigger notification to the Assignee and the group manager on time elapsed such as 50%, 75%, etc. But those notifications have not been turned on. Additionally user can manually escalate any incident ticket. Open the incident ticket -> click on top left 3 bars (just beside the incident number on the header) -> click Escalate with necessary business justification.

    What if no one responds to the incident within the SLA timeframe?

    If no one responds with the SLA timeframe, then the corresponding SLA will breach

    What is the difference between Save and Update?

    The basic functionality is the same. Save will allow you to remain on the same screen, but update takes you back to the previous screen but update takes you back to the previously accessed SNOW page after saving the ticket.

    When are configuration items (CI) required to be entered?

    The CI is not mandatory at the time of incident task creation but is mandatory when closing the incident task but it is mandatory before closing the incident ticket.

    When a user creates an incident template, is it available to only themself or to everyone?

    The template is only available to the person who created it by default. You cannot publish it for everyone or cannot mark it as global

    Is it possible to share a template with everyone and if so, how can this be done?

    Yes. The templates can be shared to groups and individuals but not globally.

    After an incident is resolved, how many days is it an option to reopen?

    Incident will be set as "Closed" after 14 days of its resolution. BMC setting was set to 30 days.

    Can users add groups to the watch list?

    A group can be add if that group has a distribution list/email. You cannot add the group names as part of the watch list.

    Is there a limit to the number of records that can be exported?

    The limit is 10k records.

    What is the purpose of a child incident?

    If there are multiple users reporting the same issue, you are able to associate incidents to one incident and call it the 'Parent' and call the earliest reported incident ticket as ‘Parent’. The updates added to the parent incident will cascade to child incidents. That way, the support users are not having to manage multiple incidents for the same issue. When you resole the Parent incident it will resolve the children.

    Can a user change the priority of an incident?

    No, they will have to change the impact or urgency which will then drive the priority.

    How can I track the number of reopened tickets?

    The user can create a report by using the filter to find how many have been reopened that day, week, month etc.

    If a ticket is transferred a different queue while in progress, will the SLA clock restart?

    The ticket will assume the corresponding SLA defined for that particular group it has been transferred to. The Response SLA will re-start but the Resolution SLA will not. Additionally, the previous SLA might be updated as “Cancelled” or “Closed” based on the SLA definition for the team.

    What if I can’t find my configuration item?

    Select “CI not found” and this will allow you to continue forward with the incident.Please report the configuration item gap to CMDB team and get the CI record updated in CMDB.

    What if a configuration item does not apply for my work area (i.e. AV technology)?

     Select “CI not applicable” /”CI not found” and ensure you complete the categorization fields.

    What if I don’t know the device but the application, technical, business service?

    You can select those services and if any application has a defined DBM record that will be synced to ServiceNow. For example, the application service for SharePoint is called “Microsoft SharePoint 2016 2016 - Production - APP-7421”.

    Can I filter on configuration items?

    Yes, you can open the configuration item display and filter upon multiple fields (class, location, manufacturing, etc..) on a configuration item.

    When is category/subcategory required?

    During the incident ticket lifecycle. As a best practice, update the fields when the incident ticket is created. This ensures an early description of the incident can be captured for incident auto-routing and/or stewardship. *Note – Moogsoft, ebonding, and/or API connections will not be required to populate these required fields on day 1. Overtime we will enable policies to enforce

    How does the category selection work?

    When you select a category, a sub-category list will appear to pick the more granular subcategory.

  • Major Incident Management

    Who are the Major Incident Managers?

    The Major incident managers are the EOC group who will review and approve/reject the major incident proposals.

    Are service organizations expected to propose an incident to a major incident and contact EOC and hence EOC handles the rest of the process?

    That is correct. Once it's acknowledged as major incident, then the communication tasks are updated and triggered by the EOC.

    What are the required fields to a propose major incident?

    Only High or Critical (P1 or P2) Incidents can be proposed as Major. Outage (CI Unavailability in BMC) and Affected Location (Areas in BMC) are mandatory to RESOLVE a Major Incident, but incident can be proposed and promoted (accepted) without this information, but it would be desirable to have them.

    Is the ServiceNow outage record similar to CI Unavailability in BMC?

    That is correct. In BMC they are known as CI Unavailability, in SN they are Outages for that CI.

    Is there a difference in the SLA clock between Major and non-Major Incidents?

    There is no difference on SLA clock. SLA for P1 and P2 Priority stays the same whether it is considered a MI or not.

    Does an incident have to be 'Proposed' first for it to be 'Promoted' by the major incident managers (MIM)? Or can the MIM directly promote it?

    Major Incident Manager can also create a major incident directly without having to wait for a proposal to promote.

    Who receives the automatic notifications when an outage is created for a Major Incident?

    Everyone in team (Assigned Group) and Watchlist participants will receive an outage notification whether it is a planned or an unplanned outage. Unless the user has opted out of receiving such notifications, they are sent to everyone.

  • On-Call Scheduling

    Can a group manager delegate the on-call scheduling to someone else?

    A group manager can setup "Shift managers" from within the group membership who can manage that going forward, but the Shift Managers must be individuals from within that group.

    Will the group manager of every group be able to set up the on-call for their support group?

    TGroup Managers will be able to set up On-Call.

    Is the ability to setup monthly rotations available?

    The tool doesn’t have monthly rotation. There are daily or weekly rotations.

    Can a user override who was scheduled for a shift?

    Only the Group Manager can override.

    Can you add more than one phone number to an on-call schedule?

    The number that is listed in the user profile is used.

  • Problem Management

    Does an incident record need to exist before a problem record can be created?

    It isn’t mandatory, but you have option to link the incident to the problem record.

    Are problem tickets ebonded?

    No, problem records will not be e-bonded.

    Can users only create the problem records and assign it to the groups that they are part of?

    Yes, that is correct.

    Are you required to create a problem ticket every time an incident occurs?

    Creating a problem record is required for Major Incidents. For all other Incidents, it is not mandatory, though encouraged.

    In ITSM we create work orders for a permanent fix for a recurring issue. Will we create incident tickets moving forward?

    To implement a permanent fix for an issue, user should raise a Normal Change (or Emergency Change – only if the issue is in progress and a high/critical priority incident is tagged). In the change record, user can relate the incident ticket under “Related Incidents” or vice versa (in incident ticket – change can be tagged as Fix Change Request). For Recurring issues, a problem ticket should be created and a change to resolve the issue should be mentioned in the problem ticket.

    Will the sequence of events in ServiceNow replace the Sequence of Events in the Major Incident template?

    That is the intent with which that functionality was built in ServiceNow. It will allow users to export that sequence of events in a report as well.

    In BMC you can create a problem record from an incident and some information is copied. (CIs, notes, etc). When you create a related problem from an incident is anything copied?

    The information that gets copied over is CI, Service, Priority, Short Description, Description and Coordinator Group.

    If users need to input impacted applications, would they use the affected CIs in that case?

    The problem ticket has 2 fields at the bottom of the page – Impacted Services and Affected CIs tabs. For Applications that were impacted by the issue, will be mentioned under Impacted Services. For Cis that were affected not only by the issue but also maybe during investigation or post resolution implementation, those CIs will be updated under Affected CIs tab. Since the Impacted Services addition is only possible through Unique IDs of CIs; current practical approach can be an update to the Affected CIs with the necessary information.

    Does anything need to be done to the existing PBIs that are still active? Will they need to be recreated?

    If they are still active and being pursued in ITSM, then they will have to be recreated in ServiceNow.

    During the assess stage you can add multiple support group approvals.

    If a user relates an incident under related incident tab, that means that the change should resolve this Incident, the incident will not be resolved automatically?

    Who can create standard changes?

    How can you add multiple CIs to a change using Excel?

    Once the Risk Questionnaire is complete, can you manually edit the risk level?

    Yes as long as the CHG has not yet been approved you can edit your answers and that could adjust your risk level.

    Can the 'Configuration Item' be blank?

    Is the CAB mandatory for all teams?

  • Service Request Management

    People are using work orders and adhoc approvals in BMC today. Can you share what that looks like in SN and where to add approvals?

    Approvals for the RITM are included in the flow designer for the workflow. Which will automatically create the approval tasks corresponding to the RITM. we have so far not enabled ad-hoc approval process for the RITMs.

    If an approver is added at the request (RITM) level, how do you know that the approver has received the approval request?

    Notifications for approvers are automatically enabled in production for all ServiceNow users, if the approver didn't disable it on his/her profile it will be received via e-mail. If the approver says no e-mail was received inform L1 ServiceNow Admin team to check.

    What capability in ServiceNow replaces the work order functionality in BMC?

    Service Requests(RITMs).

    Does the parent request of a work order correspond to the Requested Item (RITM) in ServiceNow

    Yes, that is correct.

    When does a service task (SCTASK) get created? Does it wait until the approval is granted?

    The SCTASK is created after the approval process is completed. Some items may not require approval hence the SCTASK will be created along with the service request (RITM).

    Can approvals for service requests be done from the portal?

    Approvals for Service Requests can be done from the portal . Please visit Approvals

    Can an incident be generated from an RITM that has been created in the last 30 days and is in closed status?

    An incident can be created from a closed RITM.

    Can a Service Request (RITM) be reopened after its completion?

    No, it’s not possible to reopen a RITM. The alternative is to create an incident via Service Portal which will be linked to it.

    Is it possible to manually add more approvers?

    Yes, EMIT users with access to native view may add additional approvers either at Service Request (RITM) or Service Task (SCTASK) level.

    Is it possible to remove / replace approvers after the approval request was sent?

    No, it’s not possible to do it due to audit purposes. The best approach is to cancel the request and open a new one with the correct approver(s).

    Is it possible to link a Service Task (SCTASK) to another Service Task?

    No, it’s not possible, as SCTASK is the lowest hierarchical level records in ServiceNow.

    How do I know what happened with BMC form?

    You can check mapping spreadsheet and search for the BMC form you’re looking for. It may have been migrated with the same name, renamed or even retired . Link to Mapping Spreadsheet

    How can I see all requests created by me?

    You can visit Service portal and check it under “My Requests”

    Can I see requests created by others?

    Only users with access to native view (i.e. users with ITIL or Business Stakeholder roles) may see the status of other requests.

    My Service Request (RITM) is not assigned to anyone, how can I check its status?

    Usually RITMs are not assigned to individuals / analysts, only SCTASKs. You can check if there are any SCTASKs related to the RITM on “My Requests” session on Service Portal, selecting the RITM you want and check the tab “Associated Tasks”

    Which form should I use to create a generic request, like “Create Work Order” in ITConsole?

    You can goto Service Portal and you will find Generic Service Request

    NOTE: the submitter must know which assignment group is responsible to work on it.

    I’ve received a link to a form but I can’t access it.

    The forms are categorized in two different catalogs: Service Catalog, open to all users, and IT4IT catalog, which contains forms to be used exclusively by users who are part of an assignment group (i.e. users with ITIL role).
