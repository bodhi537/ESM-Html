Begin Your Citizen Developer Journey

Step 1

Identify Your Use Case

The first step in your citizen developer journey is to identify the right use cases. The ServiceNow low code platform has many capabilities but it is important to fully understand the process you are attempting to digitize. Your access to the Citizen Developer environment allows you to:

  • Create catalog items and forms
  • Create flows
  • Create tables (requires review and approval)
  • Create automations
  • Create monitoring alert rules to enable event management

After identifying the Use Case, please submit a Solution Review Request. Please refer to Step 6:line item 1 for more information. A solution review can take up to 10 business days and it would be a good idea to get it started as early as possible. However, as a Citizen Developer your developer journey can proceed in parallel to Step 2.

Step 2

Review Guardrails

Clear and concise guardrails allow you to be in alignment with our objectives and expectations. These guardrails provide you with a degree of independence within a safe and secure operating space.

      1. Every solution has a defined owner and approved development track
      2. Submit all solutions thru the ITOE design review
      3. Leverage EA Mandates for modern authentication methods (link)
      4. Ensure documentation (design and test plans) exists
      5. Align with UI/UX Best Practices (link)
      6. Align with ITIL Best Practices (incident, problem, change management) (link)
      7. Ensure code does not impact performance
      8. Use approved developer tools (e.g., Flow Designer versus Workflow)
      9. Develop solution within correct Application Scope (not ‘Global’)
      10. Align with coding best practices (link)

For full list of guardrails, access goto/Guardrails

Step 4

Request Access

To request access to Citizen Development server submit an Add/Remove users to ITSM Support Group request:

  • Select Citizen Development in the Selct Group field
  • Select yourself in the Select Users to Add field
  • Once you receive access to the environment use goto/emcdev to access it.
  • Your Acceptance(goto/emacc) and Production (goto/emprod) access will allow you to test and validate forms and flows. No development work should be done in these environments. If something needs to be fixed, fix it on the Citizen Development instance.


Step 5

Get Help

ServiceNow is a powerfull platform and has many capabilites. If you run into issues with your application there are ways you can get help.

  • Google is your friend when it comes to questions related to ServiceNow development. There is a huge developer community out there with answers to most common issues
  • Join the ServceNow Citizen Developer CoP Zoom channel. If you cannot find answers to your question online, then post your question in this chat. We have a lot of developers with experience in this channel and someone will answer your question
  • If you cannot find an answer in the Zoom channel, then the next step would be to contact Citizen Developer lead Kithsiri Wijewardena(Kith). He has access to ServiceNow support personnel and will definitely find a solution to your problem
  • Join our weekly Community of Practice meeting held on Thursdays. Kith will discuss a topic related to ServiceNow development and help you find answers to your questions. You can find these meeting in EventX. Search for ServiceNow Citizen Developer and sign up. There are two sessions : AP and AM/EU

Step 6

Build Your Application

The following steps explain the process that every ServiceNow developer should follow.

  • After identifying a process to automate submit a ESM Solution Review Request (goto/esm_srr).
  • Approval process can take upto 10 business days. You can start working on the application after submitting a Solution Review Request
  • Please note that an approved Solution Review Request is needed before your application can go to ACC
  • You can check if your solution review request is approved at goto/esm_srrt. You need the Solution Review RITM to find your request.
  • If you would like to add additional developers to your application please submit a Generic Service Request (goto/gsr) to L1 ServiceNow Admin team. Please provide the application name and the developer name.
  • Since we have montly copy-backs, its very importnt to back up your application to GitHub. Please follow this document to connect your application to GitHub.
    • After completing the development work and testing the form, publish the application to My Application Repository (App Store).
    • Submit a ESM Code Deoployment Request (goto/esm_cdr) to install your application in Acceptance.
    • A Solution Review RITM number is needed and the Solution Review Request must be approved.
    • After submitting a Code Deployment Request, a Design Review will be done. After the Design Review is done, your application will be promoted to ACC.
    • You can check if the design review is complete at goto/esm_srrt. You need the Solution Review RITM to find your request.
      • Deployment Status = Design Review: Design review is pending
      • Deployment Status = Ready for Accept Migration: Design review has been completed and it's ready for ACC migration
      • Deployment Status = Back to Developer: Design review has been completed and the developer is required to fix something prior to ACC migration
  • After getting the application installed in Acceptance, perform User Acceptance Testing (UAT)
    • Identify the users that would test your application in ACC
    • You need a confirmation email from the testing team stating that all tests are successful.
  • If the testing team identify any issues, they have to be fixed in CitizenDev and publish a new version to App Store and then get it installed in ACC.
  • After completing UAT, create a Change Ticket
    • Select one of the following values in the Change Template field and it should populate most fields
      • ServiceNow Change - Application Store
      • ServiceNow Change - Update Sets
    • Make sure the change ticket contains valid Implementation, Backout and Test plans.
    • Use this ticket as an example: CHG0056476
    • Confirm that the Configuration Item field is set to ServiceNow ITSM SaaS - Production - APP-4181. This is how you get an invitation to the CRB meeting.
    • It is also very important to link your Solution Review RITM to the change ticket, in Requested Item tab below Related Links.
  • Present your solution in the CAB meeting
    • Be prepared to present your solution in the CAB meeting. These meetings are held on Wednesday mornings.
    • If you don't get an invite to a CAB meeting, please contact Rutuja Jagdale.
  • After the CAB meeting, your solution is ready for PROD migration. PROD migrations happen on Thu & Fri.
    • After the PROD migration you need to do a quick test to make sure the application is working as expected.
    • If you find any issues, you can work with the L1 ServiceNow Admin team to resolve them.
    • You also can uninstall the new version and go back to the old version. This option is only available for 24hrs after installing a new version. Therefore it's important to test your application in PROD promptly.
